# TCP Message Project - מערכת הודעות מבוססת TCP
פרויקט זה מדגים מימוש של מערכת תקשורת מסוג שרת-לקוח (Client-Server) מעל פרוטוקול TCP/IP, המאפשרת העברת הודעות טקסטואליות בזמן אמת.

המערכת בנויה להתמודד עם חיבורים מרובים ולנהל תעבורת מידע אמינה בין הצדדים.

**הערות:** 
   1. הדו"ח המסכם מצורף כקובץ PDF (יש לשים לב לכפתור More pages עבור טעינת המשך הדפים בדו"ח).
   2. הקובץ packets-first-part.pcap שייך לחלק הראשון.
   3. הקובץ packets-project-second-part.pcap שייך לחלק השני.
   
## טכנולוגיות וכלים

הפרויקט נכתב בשפת **Python** ועושה שימוש בספריות הסטנדרטיות (Standard Library) ללא צורך בהתקנות חיצוניות. הטכנולוגיות המרכזיות כוללות:
* **שפה - Python 3**.
* **ספרייה - Socket API** המשמשת ליצירת נקודות קצה (Endpoints) לתקשורת, ניהול ה-Handshake של TCP, ושליחת/קבלת בייטים של מידע.
* **ספרייה - Threading (Multi-threading)** משמשת (בצד השרת) לניהול מספר לקוחות במקביל. כל לקוח שמתחבר מקבל "תהליכון" (Thread) ייעודי המאזין להודעות ממנו, מה שמונע חסימה של השרת הראשי.
* **פרוטוקול TCP** משמש להבטחה ואמינות, סדר ושלמות הנתונים המועברים.


## תכולת הפרויקט ומבנה הקוד

הפרויקט מחולק לשני חלקיפ עיקריים:

### 1. צד השרת (`server.py`)
השרת הוא "המוח" של המערכת. תפקידיו העיקריים:
* **האזנה (Binding & Listening):** פתיחת Socket והמתנה לחיבורים נכנסים בכתובת IP ופורט מוגדרים.
* **ניהול לקוחות:** קבלת חיבורי לקוחות (Accept) ושיוך כל חיבור ל-Thread נפרד, תומך בלפחות חמישה לקוחות.
* **עיבוד הודעות:** קבלת הודעות מהלקוחות, פענוחן (Decoding) והדפסתן (או הפצתן לשאר הלקוחות, בהתאם ללוגיקה הפנימית).

### 2. צד הלקוח (`client.py`)
הלקוח הוא ממשק המשתמש. תפקידיו העיקריים:
* **יצירת קשר:** ייזום חיבור TCP מול השרת.
* **ממשק קלט:** קבלת טקסט מהמשתמש דרך הטרמינל.
* **שליחת נתונים:** קידוד ההודעות (Encoding) ושליחתן לשרת.

## הוראות התקנה והרצה

מכיוון שהפרויקט מתבסס על ספריות מובנות של Python, תהליך ההתקנה פשוט ומהיר.

### דרישות מוקדמות
* Python גרסה 3.6 ומעלה.

### שלב 1: הורדת הפרויקט
תעתיקו את הקוד הבא לטרמינל:
```bash
git clone https://github.com/YasharPa/TCPmessageproject.git
cd TCPmessageproject
```

בנוסף, מכיוון שהפרויקט משתמש בממשק גרפי מתקדם, יש להריץ את פקודת ההתקנה הבאה לפני הרצת הקוד:
```bash
pip install customtkinter
```

#

   ### שלב 2: הפעלת השרת (Running the Server)
השרת חייב לפעול ראשון כדי להאזין לבקשות התחברות.

1. פתחו חלון טרמינל (Terminal/CMD).
2. נווטו לתיקיית הפרויקט (`cd TCPmessageproject`).
3. הריצו את הפקודה:

```bash
python server.py
```
אם הפקודה הזו לא עובדת/מריצה, נסה את השורה הבאה:
```bash
py ./server.py
```

**אימות:** הטרמינל יציג אינדיקציה שהשרת מאזין (**Listening**). **אין לסגור חלון זה.**

### שלב 3: הפעלת הלקוח (Running the Client)
ניתן להפעיל לקוח אחד או יותר במקביל.

1. פתחו חלון טרמינל **חדש ונפרד** (אל תשתמשו בחלון של השרת).
2. נווטו לתיקיית הפרויקט (`cd TCPmessageproject`).
3. הריצו את הפקודה:

```bash
python client.py
```
אם הפקודה הזו לא עובדת/מריצה, נסה את השורה הבאה:
```bash
py ./client.py
```
**שימוש:**
* לאחר החיבור, ניתן להקליד הודעות בטרמינל וללחוץ `Enter` לשליחה.
* ההודעות יתקבלו ויוצגו בחלון השרת (או אצל לקוחות אחרים, בהתאם ללוגיקה בקוד).
* כדי לחבר משתמשים נוספים, פתחו טרמינלים נוספים וחזרו על שלב 4.

##  פתרון תקלות נפוצות

* **תקלת Connection Refused:** שגיאה זו מעידה לרוב שהשרת אינו רץ, או שהלקוח מנסה להתחבר לפורט/כתובת שגויים. ודאו ששלב 3 בוצע בהצלחה.
* **תקלת Address already in use:** הפורט תפוס. אם סגרתם את השרת וניסיתם להפעיל מיד מחדש, המתינו מספר שניות או שנו את מספר הפורט בקוד (בשני הקבצים).
* **תקלת Firewall:** בעבודה בין מחשבים שונים, ודאו שחומת האש אינה חוסמת את הפורט הנבחר.
